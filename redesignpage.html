Redesign

PROBLEM: How can we identify usability problems on a website, and what design techniques can we implement to improve the website?

In this project, I experienced the workflow of redesigning a responsive website through identifying existing usability problems that can be fixed with better UI design. I chose to redesign the Taipei Metro website, as this is a site I frequently visit for information about the metro. The process included the following steps:

  1. Identifying usability problems in the original website and understanding the website's accessibility
  2. Creating low-fidelity wireframes using Balsamiq for 3 different screen sizes
  3. Creating high-fidelity wireframes using AdobeXD for 3 different screen sizes
  4. Creating a visual design style guide to standardize the website's interactive elements
  5. Redesigning the website into a responsive and improved version

KEY TAKEAWAYS

  • This was my first taste at designing a website from scratch, and I learned that this is a complex and nuanced process with lots of elements for consideration!
  • Identifying usability problems is a key step in understanding how to improve a website, and small design changes can drastically affect the flow of a page
  • Low fidelity and high fidelity wireframes can provide a lot of insight into the functionality of websites. Putting a lot of thought into each element can help with the coding process later.
  • We often take the responsiveness of websites for granted, but considering how each element will change with responsive design is really crucial in ensuring a good user experience!
